# Dialog Emotion Timeline

Gradio demo for visualising an emotional probability distribution across a
dialogue: `joy`, `warmth`, `sadness`, `anger`, `anxiety`, `surprise`, `neutral`
(the 7-class scheme used by the trained models).

**This repository is the demo, and only the demo.** It reads a ready-made CSV and
draws it — emotional trajectories, per-sender stats, per-message breakdowns, with
softmax-temperature and smoothing controls. Producing that CSV (Telegram export →
parse → emotion model → CSV) lives in the sibling
[`dialog-emo-models`](../dialog-emo-models) project; nothing in here loads a model
or scores a message.

## Local run

```bash
uv run python app.py
```

Set `DIALOG_CSV` to point the default view at any compatible CSV (otherwise the
bundled `data/synthetic_dialog.csv` is used). You can also upload a CSV from the UI.

## CSV contract (what the UI reads)

Columns, in order, validated by `dialog_emo_demo.schema.validate_dialog_frame`:

```
turn_index, timestamp, sender, text, joy, warmth, sadness, anger, anxiety, surprise, neutral
```

Per row: each emotion in `[0, 1]`; the seven emotions sum to `1 ± 1e-3`. Bad rows
fail loudly with `DialogDataError`. The loader sorts by `turn_index`; the UI then
applies the softmax-temperature and smoothing controls on top of these values.

Any tool that emits this shape works — the canonical producer is
`dialog-emo-models` (`uv run dialog-emo ...`), which scores a Telegram export with
one of its trained checkpoints and writes exactly these columns.

## Bundled data

- `data/synthetic_dialog.csv` — the default view; a small hand-authored sample,
  regenerated by `scripts/generate_synthetic_data.py` (no model involved).
- `data/dialog_misha_evelina_emotions.csv` — a real dialogue scored by
  `dialog-emo-models` (`rubert-tiny2-finetune`), kept as a realistic upload example.

```bash
uv run python scripts/generate_synthetic_data.py   # rewrite the synthetic sample
```

## Modules

| module | role |
| --- | --- |
| `dialog_emo_demo/schema.py` | CSV contract: load / validate / temperature / sender filtering |
| `dialog_emo_demo/plotting.py` | Plotly figures + the emotion palette and labels |
| `dialog_emo_demo/ui.py` | Gradio layout, stats card, message cards |
| `dialog_emo_demo/synthetic.py` | hand-authored sample dialogue |

## Tests

```bash
uv run pytest
```

Covered: CSV schema (loading, validation, temperature, sender filtering), plotting
(traces per emotion, area mode, slice bars), and the app/UI rendering.
